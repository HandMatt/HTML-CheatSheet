Main root

<!DOCTYPE html>

All HTML documents must start with a <!DOCTYPE> declaration. The declaration is not an empty HTML tag. It is an "information" to the browser about what document type to expect.

<html> ... </html>

The <html> element represents the top-level element of an HTML document. Referred to as the root element. All other elements must be decendants. You should always include the 'lang' attibute inside the <html> tag, to declare the language of Web page. This will assist search engines and browsers.

<head> ... </head>

The <head> element contains machine-readable information (metadata) about the document. Information such as; title, scripts , and style sheets.

<body> ... </body>

The <body> element represents the content of an HTML document. There can be only one <body> element in a document.

Example:

<!DOCTYPE html>
<html lang="en">
  <head>...</head>
  <body>...</body>
</html>

Document metadata

<meta/>

The <meta/> tag defines metadata about an HTML document. Metadata is data(information) about data. Typical uses are; page description, author, published date, keywords, character set and other page information.

<base/>

The <base/> tag specifies the base URL and/or target for all relative URL's in a document. The tag must either have an HREF or target attribute present, or both. There can only be one single <base/> element in a document, and it must be inside the <head> element.

<title> ... </title>

The <title> element defines the document's title that shows in a browser's title bar or a page's tab.

<link/>

The <link/> element, used to create relationships with external pages or documents. This element links to stylesheets and site icons ("favicon"), among other things.

<style> ... </style>

The <style> element contains style information for a document, or part of a document.

<script> ... <script/>

The <script> element includes all scripting information, or links to external scripts. This element can also be included within the body to dynamically generate content.

Example:

<html>
  <head>
    <meta charset="utf-8">
    <base href="http://HandosHTMLCheatsheet.com" target="_blank" />
    <title>Hando's HTML Cheatsheet</title>
    <link href="./resources/css/styles.css" type="text/css" rel="stylesheet">
    <script type="text/javascript">
      var myVar = 0;
    </script>
  </head>
  <body>...</body>
</html>

Tables

Table Tags
Tag Name Description
<table> Table The wrapper element for all HTML tables
<caption> Caption Specifies the caption (or title) of a table, and if used is always the first child of a table.
<thead> Table Head The set of rows defining the column headers in a table
<tbody> Table Body The set of rows containing actual table data
<tfoot> Table Foot The set of rows defining the footer in a table
<tr> Table Row The table row container
<td> Table Data The table data container
<th> Table Head The data containers for the table head
Table Attributes
Attribute Name Description
colspan Column Span Defines how many columns a td element should span
rowspan Row Span Defines how many rows a td element should span

Example:

<table>
  <caption>I am Table</caption>
  <thead>
    <tr>
      <th colspan="2">The table header</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>The table body</td>
      <td rowspan="2">and two columns</td>
    </tr>
    <tr>
      <td>with two rows</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td colspan="2">The table footer</td>
    </tr>
  </tfoot>
</table>

Output:

I am Table
The table header
The table body and two columns
with two rows
The table footer

Forms

<form> ... </form>

The <form> element represents a document section that contains interactive controls for submitting information to a web server.

<fieldset> ... </fieldset>

The <fieldset> element is used to group several controls as well as labels within a web form.

<legend> ... </legend>

The <legend> element represents a caption for the content of its parent <fieldset>

<label> ... </label>

The <label> element represents a caption for an item in a user interface.

<input>

The <input> element is used to create interactive controls for web-based forms in order to accept data from the user. A wide variety of data input types and control widgets are available, depending on the device and user agent.

<textarea> ... </textarea>

The <textarea> element represents a multi-line plain-text editing control. Useful when you want to allow users to enter a sizeable amount of free-form text. For example a comment on a review or feedback form.

<datalist> ... </datalist>

The <datalist> element contains a set of <option> elements that represent the values available for other controls.

<select> ... </select>

The <select> element represents a control that provides a menu of options.

<optgroup> ... </optgroup>

The <optgroup> element creates a grouping of options within a <select> element.

<option> ... </option>

The <option> element is used to define an item contained in a <select>, an <optgroup> or a <datalist> element. <option> can represent menu items in popups and other lists of items in a HTML document.

<button> ... </button>

The <button> element represents a clickable button. This can be used in forms or anywhere in the document where simple button functionality is needed.

<progress> ... </progress>

The <progress> element displays an indicator showing the completion progress of a task, typically displayed as a progress bar.
